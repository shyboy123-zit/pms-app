# 📊 PMS 시각화 개선 구현계획서 (Visual Access)

> 목적: **대시보드 외 다른 페이지에서도** 숫자·표를 "한눈에" 빠르게 인지하도록 시각적 요소(게이지·색상·도넛·미니차트·상태타일)를 확장 적용한다.
> 작성: 2026-06-06 · 대상: `pms-app` 전 페이지

---

## 1. 배경 & 현재 상태

| 구분 | 현재 |
|---|---|
| 차트(recharts) | **매입매출(Sales) 단 1곳** |
| KPI 카드 | Materials·Products·WorkOrders·Equipments·DailyProduction·InventoryInOut·Delivery·WorkHistory (숫자 카드 위주) |
| 대부분 페이지 | **데이터 테이블 중심** — 색상 배지 정도만 있고 시각적 패턴 약함 |
| 대시보드 | ✅ 호기 배치도·게이지·도넛·상태타일·카드모달 완비 (이번에 구축) |

**핵심 아이디어**: 대시보드에서 만든 시각 패턴(게이지/도넛/상태타일)을 **공용 컴포넌트로 추출**해 다른 페이지에 재사용한다. 새로 만드는 비용보다 재사용 비용이 훨씬 적다.

---

## 2. 설계 원칙

1. **색상 일관성(Semantic Color)** — 전사 통일된 의미색 사용
   - 🟢 정상/가동/완료 = `--success` · 🟡 주의/대기/임박 = `--warning` · 🔴 위험/정지/지연/불량 = `--danger` · 🔵 정보/진행 = `--info` · ⚪ 비활성 = `--text-muted`
2. **글랜서빌리티(Glanceability)** — 숫자를 읽기 전에 **색·길이·모양**으로 먼저 파악되게
3. **점진적 정보 공개(Progressive Disclosure)** — 요약(카드/게이지) → 클릭 시 상세(모달/표). 대시보드 카드모달 패턴 그대로
4. **모바일 우선 분기** — 좁은 화면에선 표 대신 카드/게이지, 버튼+팝업 (배치도에서 검증된 방식)
5. **테이블은 유지, 시각은 추가** — 기존 표를 없애지 않고 행 안에 게이지/색상셀을 얹거나 상단에 요약 시각 블록을 추가

---

## 3. 공용 시각화 컴포넌트 (Phase A — 기반 구축)

> 한 번 만들어 전 페이지 재사용. `src/components/viz/` 폴더 신설.

| 컴포넌트 | 용도 | 재사용처 |
|---|---|---|
| `LevelGauge` | 수평 채움 게이지(현재/기준) + 임계 색상 | 재고수위, 진척률, 수명타수, 달성률 |
| `DonutKpi` | 도넛 %+중앙 라벨 (배치도 가동률 일반화) | 가동률, OK/NG, 미수/미지급 비중 |
| `StatusTile` | 색상 상태 타일/배지 통일 | 설비·작업·금형·발주·품질 상태 |
| `Sparkline` | 표 셀 안 초소형 추세선 | 거래처별·제품별 추이 |
| `MiniBar` | 소형 막대(파레토/비교) | 불량유형, 거래처 잔액, 부서 인건비 |
| `HeatCalendar` | 일자별 색농도 히트맵 | 일일 생산량/불량/근태 |
| `KanbanBoard` | 상태별 컬럼 보드 | 작업지시, 구매발주 |
| `MiniTimeline` | 세로 타임라인 | 금형 이력, 지원사업 일정, 감사로그 |

색상 토큰은 이미 `index.css`에 존재(`--success/--danger/--warning/--info`). 누락분만 보강.

## 4. 페이지별 적용 계획 (우선순위순)

### 🥇 1순위 — 효과 大 (현장 즉시 체감)

#### ① 설비관리 (Equipments)
- **설비 상태 보드**: 표 위에 설비를 색상 타일 그리드로 (가동중🟢/대기⚪/점검🔴). 대시보드 `FactoryFloorMap`의 머신 카드 컴포넌트 재활용.
- 상단 **가동률 도넛** + 가동/대기/정지 카운트
- 컴포넌트: `StatusTile`, `DonutKpi`

#### ② 작업지시 (WorkOrders)
- **칸반 보드**: 대기 / 진행중 / 완료 / 취소 4컬럼, 각 카드에 **진척 게이지**(produced/target)
- 토글로 표↔칸반 전환
- 컴포넌트: `KanbanBoard`, `LevelGauge`

#### ③ 품질관리 (Quality)
- **불량 추세 라인차트**(일별 불량률) + **불량유형 파레토 막대** + **OK/NG 도넛**
- 표 행의 판정(OK/NG)을 색상 강조
- 컴포넌트: recharts LineChart, `MiniBar`, `DonutKpi`

#### ④ 원재료관리 (Materials)
- 각 행에 **재고 수위 게이지**(현재 vs 안전재고, 미달 시 빨강)
- 상단에 안전재고 미달 비율 도넛 (KPI 카드는 이미 있음 → 시각 보강)
- 컴포넌트: `LevelGauge`, `DonutKpi`

### 🥈 2순위 — 효과 中

#### ⑤ 금형관리 (Molds)
- **금형 수명 게이지**(누적 타수 / 한계 타수) — 색상으로 교체 임박 경고
- 점검 **D-day 색상 배지**(90일 경과 임박/초과)
- 금형별 이력 **세로 타임라인**(수리/출고/입고)
- 컴포넌트: `LevelGauge`, `StatusTile`, `MiniTimeline`

#### ⑥ 제품관리 (Products)
- 행에 **재고 수위 막대**(현재고/안전재고), 상태 색상
- 컴포넌트: `LevelGauge`

#### ⑦ 일일작업현황 (DailyProduction)
- **호기별 일일 달성 게이지**
- **생산 캘린더 히트맵**(월별 일자별 생산량 색농도)
- 컴포넌트: `LevelGauge`, `HeatCalendar`

#### ⑧ 매입매출/결제 (Sales / PaymentTab)
- 이미 추이 차트 있음 → **거래처별 잔액 막대그래프**, **미수/미지급 도넛**, 결제 진행 게이지 추가
- 컴포넌트: `MiniBar`, `DonutKpi`, `LevelGauge`

### 🥉 3순위 — 효과 小 / 선택

| 페이지 | 적용 아이디어 | 컴포넌트 |
|---|---|---|
| 구매관리(Purchase) | 발주 상태 칸반/색상 배지 | `KanbanBoard`, `StatusTile` |
| 급여관리(Payroll) | 월별 인건비 추이 막대 + 부서별 비중 도넛 | `MiniBar`, `DonutKpi` |
| 국가지원사업(GovernmentSupport) | 사업 일정 타임라인 + D-day 색상 | `MiniTimeline`, `StatusTile` |
| 거래처관리(Suppliers) | 거래처별 거래액 막대 | `MiniBar` |
| 직원관리(Employees) | 부서별 인원 도넛, 근속분포 | `DonutKpi`, `MiniBar` |
| 감사로그(AuditLog) | 작업유형 분포 + 타임라인 | `MiniBar`, `MiniTimeline` |
| 입출고관리(InventoryInOut) | 품목별 입출고 추이 스파크라인 | `Sparkline` |

## 7. 리스크 & 주의

1. **거대 페이지 파일** — Materials(1,319)·Sales(1,299)·InventoryInOut(1,207)·Employees(2,007)는 수정 시 회귀 위험. 시각 블록은 **상단 요약/행 셀 추가** 위주로 최소 침습.
2. **성능** — 행마다 게이지/스파크라인은 가벼운 CSS/SVG로 구현(차트 라이브러리 남발 금지). 큰 목록은 상단 요약만 차트.
3. **모바일** — 배치도에서 검증된 "카드+팝업" 패턴 재사용. 표는 가로 스크롤 대신 카드 전환.
4. **다크모드** — 색상은 반드시 CSS 변수 사용(하드코딩 #색상 자제), 대시보드 위젯 호환 기준 준수.
5. **데이터 전제** — 게이지/소진예상은 제품 중량·캐비티·안전재고 입력이 채워져야 정확. 미입력 시 "정보 없음" 처리.
